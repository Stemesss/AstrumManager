# Архитектура AstrumManager

Эта папка фиксирует текущее устройство проекта на Этапе 1 без изменения бизнес-логики.

## Состав документации

- `overview.md` — общая карта системы, точки входа и runtime-сценарии.
- `components.md` — компоненты Python-бота, Node.js workspace, модели и хранилища.
- `contracts.md` — действующие контракты между слоями и инварианты, которые нельзя ломать.
- `known-issues.md` — найденные архитектурные проблемы и технические ограничения без исправлений.

## Границы этапа

- Документация описывает текущее состояние репозитория.
- Python-бот в `main.py` остаётся основной бизнес-системой.
- Документы не являются планом рефакторинга и не изменяют существующее поведение.
