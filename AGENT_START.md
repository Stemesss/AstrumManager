# AGENT_START.md — Главная точка входа агента

> Читать ПЕРВЫМ перед любой работой над проектом AstrumManager.

---

## ОБЯЗАТЕЛЬНЫЙ ПОРЯДОК ЧТЕНИЯ ПАМЯТИ

Перед началом любой работы выполни следующие шаги по порядку:

1. `AGENT_START.md` (этот файл)
2. `replit.md` — стек, архитектура, команды запуска
3. `PROJECT_HISTORY.md` — история этапов и выполненных задач
4. `.agents/memory/MEMORY.md` — индекс всей памяти агента
5. `.agents/memory/project_state.md` — текущий статус, версия, этап
6. `.agents/memory/workflow.md` — текущий рабочий процесс и этапы
7. `.agents/memory/dev_rules.md` — правила разработки (обязательно к исполнению)
8. `.agents/memory/change_log.md` — последние изменения
9. `.agents/memory/audit.md` — найденные и исправленные дефекты

После чтения сообщи пользователю:
- текущий статус проекта
- текущую версию
- последний завершённый этап
- следующий этап

Затем остановись и жди задания.

---

## ПРАВИЛА РАЗРАБОТКИ (краткий свод)

Полные правила — в `.agents/memory/dev_rules.md`.

- Никогда не изменяй код без явного задания от пользователя.
- Каждое изменение сопровождается обновлением файлов памяти.
- После каждого завершённого задания — полный цикл сохранения (см. ниже).
- Не используй polling на Replit — только webhook.
- Всегда соблюдай структуру модулей из `.agents/memory/bot-structure.md`.
- Инъекция зависимостей — только через `dp["service_name"]` (см. `userservice-injection.md`).
- Никнеймы — только через `build_full_nick(name, role)` (см. `nick-format.md`).

## ПОРЯДОК ОБНОВЛЕНИЯ ПАМЯТИ

После каждого завершённого задания автоматически обновить:

| Файл | Когда обновлять |
|------|----------------|
| `project_state.md` | После каждого задания |
| `change_log.md` | После каждого задания |
| `PROJECT_HISTORY.md` | При завершении этапа |
| `workflow.md` | При смене этапа |
| `audit.md` | При обнаружении или исправлении дефекта |
| `replit.md` | При смене версии проекта |
| `MEMORY.md` | При добавлении нового topic-файла |

Не ждать отдельной команды — обновление памяти является частью каждого задания.

## ОБЯЗАТЕЛЬНЫЙ ЦИКЛ СОХРАНЕНИЯ ПОСЛЕ КАЖДОГО ЗАДАНИЯ

После ЛЮБОГО изменения проекта Agent ОБЯЗАН выполнить полный цикл:

1. Обновить память проекта (все релевантные файлы в `.agents/memory/`).
2. Выполнить smoke-test (проверить логи webhook — HTTP 200, нет критических ошибок).
3. Проверить запуск всех workflow (Telegram Bot / API Server / Component Preview Server).
4. Убедиться, что Replit Checkpoint создан (платформа создаёт автоматически при завершении задания).
5. Выполнить push в origin/main через `bash scripts/push.sh`.
6. Проверить, что GitHub содержит последний коммит (`git ls-remote origin main`).
7. Только после успешного push считать задачу завершённой.
8. Только после этого отправлять итоговый отчёт пользователю.

**Если push НЕ выполнен** — запрещено писать «СТАТУС: выполнено».
Необходимо указать причину, что GitHub ещё не обновлён, и что нужно сделать.
